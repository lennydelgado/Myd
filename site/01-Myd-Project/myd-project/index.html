    Myd-project¶

    Présentation du projet Myd¶

    But du projet¶

    Le projet consiste à automatiser la contenairisation d’un site web crée à l’aide de MkDocs.

    Info

    MkDocs est un outil de documentation open-source permettant très simplement de documenter rapidement un projet. Il est basée sur le language Python.

    Mes outils¶

    Durant le projet, j’ai utilisé plusieurs outils afin de mener à bien le projet, comme par exemple Docker qui constitue le cœur de mon projet. J’ai utilisé notamment Harbour qui est un service permettant de stocker dans le cloud des images Docker, à la manière du Docker Hub.

    Pour ce qui est du site web, j’ai utilisé MkDocs qui est un générateur de site web statique destiné à créer une plateforme de documentation. Le site une fois généré, a pu être déployé grâce à Nginx qui est un logiciel libre de serveur Web.

    Liste des outils

    • Docker
    • Harbour
    • MkDocs
    • Nginx

    Etapes du projet¶

    Image Python¶

    Tout d’abord, j’ai dû créer mon tout premier fichier Docker avec une image Debian (j’ai utilisé l’image debian-bulleye-slim pour avoir un container léger). Par la suite, j’ai légèrement configuré ce dernier pour le rendre prêt pour la prochaine étape.

    Note

    Inutile de préciser, mais chacun des container générer ont tous été publier sur Harbour en privé pour avoir un accès facile et pour pouvoir les utiliser commes des images.

    Ensuite, j’ai ajouté dans mon container Debian, le langage obligatoire pour l’installation de MkDocs qui est Python. Toute cette étape m’a généré une image python prête à l’emploi pour la prochaine étape.

    Image Nginx¶

    Enfin, j’ai dû utiliser cette fois ma propre image Python, j’ai par la suite récupéré depuis mon répertoire Github, mes fichiers Markdown nécessaires à la génération de ma page web.

    Après avoir installé MkDocs et toutes les dépendances nécessaires, j’ai ensuite généré les pages pour Nginx afin qu’il puisse me le traduire en page web et non en Markdown (MkDocs génère des pages HTML à partir de fichier Markdown).

    Grâce aux connaissances que j’ai acquises sur Docker, j’ai réussi dans le même fichier de configuration à récupérer cette fois-ci l’image Nginx (nginx:1.23.0-alpine, image très légère également) ce qui m’a permis par la suite à simplement avoir dans ma dernière image seulement Nginx et Alpine (Python étais nécessaire uniquement pour la génération des pages web et non le fonctionnement).

    Bilan¶

    Toutes ces étapes ont été éffectué par ce qu’on appelle des Docker files (des fichiers de configurations de container Docker) et des scripts Shell pour le lancement, permettant une automatisation de tout les procéssus décrit au dessus.

    On se retrouve avec une image final qui est légère et dans laquelle n’est présents que ce dont on a besoins.

    Info

    Le projet consisté principalement à essayer de reproduire le modèle déjà en place ce qui a permit à mon tuteur de savoir comment je me débrouillais. J’ai du faire en parti sans modèles, la partie Nginx. Ce projet était centré sur le DevOps. J’ai réussi à généraliser au maximum ce qui permet à n’importe qui voudrais de l’installer.

    Difficultés rencontrées¶

    Durant le projet, j’ai eu pas mal de difficultés en ce qui concernait Docker et ma méconnaissance des fonctionnement de MkDocs ou encore Nginx.

    La partie principale étant Docker, j’ai eu des difficultés parce que je ne l’avais presque jamais utilisé. Mais c’est aller très vite mieux, le langage est visuels et tout ce qui touche Docker est très bien documenté.

    Liste des difficultées

    • La prise en main de Docker
    • L’installation de la surcouche Nginx notamment la configuration

    Les comptétences acquises¶

    J’ai réussi à vraiment à gagner en aisance avec Docker, il m’a fallu du temps certes Docker est un outil très puissant et intéressant à manipuler.

    Je me suis aussi entraîné à documenter par exemple en écrivant des phrases dans le code, afin d’être le plus clair possible dans le fonctionnement du projet.

    J’ai compris le réel fonctionnement et utilités de Docker, mon seul projet que j’ai fait à l’école me laisser encore quelques zones d’ombres.

    Mon retour sur le projet¶

    J’ai réussi à vraiment à gagner en aisance avec Docker, il m’a fallu du temps certes Docker est un outil très puissant et intéressant à manipuler.
